scratch.git FAQ

Why does nothing happen when I commit?

You may have opened the wrong project. Make sure that the project that you opened was in the same folder as the one you chose when starting the project. In the future, this may be made less confusing. Also, make sure that the Sprite folders addon is disabled (for now).

Do I have to use a valid name or registered email to make commits?

No, just make sure that the email is formatted correctly.

How can I label script changes with something other than a number?

If you attach a text comment to the first block of a script, scratch.git will use the comment's label instead of a number.

When I enable an addon, it breaks. Why?

As of now, the following addons are not fully compatible with scratch.git:

  • Sprite folders
  • Cat blocks
  • Customizable block colors (diffs only use regular, high-contrast, and dark-mode colors)
  • Square block inputs

When I delete or detach a block from script, it selects the wrong scripts. Why?

If you save after forming two identical scripts, the wrong script may be selected due to how we process scripts. Additionally, scratch.git may also display extraneous diffs where more scripts are changed than in reality. Both of these oddities don't have a meaningful impact on development or an easy solution, so they will likely stick around.
